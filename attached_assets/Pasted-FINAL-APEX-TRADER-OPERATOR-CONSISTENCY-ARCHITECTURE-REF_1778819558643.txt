CRITICAL PRIORITY #1 — CENTRALIZED EXCHANGE STATE
=================================================

CURRENT PROBLEM:

Different areas of the UI appear to maintain DIFFERENT interpretations of:

* active exchange
* trading mode
* broker context
* account balance
* simulation/live state

EXAMPLE:

Top telemetry:
“LIVE TRADING — KRAKEN — SPOT ONLY”

Lower Activity Hub:
“KRAKEN / SIMULATION BROKER”

This creates severe operator ambiguity and indicates fragmented state management.

====================================================
REQUIRED FIX
============

Create ONE centralized authoritative exchange/trading context.

ALL dashboard components must derive state ONLY from:

* activeExchange
* tradingMode
* brokerType
* accountContext
* portfolioContext
* executionContext

NO component may:

* assume simulation independently
* assume Kraken independently
* assume paper mode independently
* maintain stale local exchange state
* display isolated mock telemetry

====================================================
GOAL

When exchange or mode changes:
EVERY component updates consistently.

This includes:

* top telemetry
* Activity Hub
* account metrics
* portfolio metrics
* AI execution layer
* positions
* balances
* fees
* trade history
* charts
* terminal feed
* exchange labels
* mode labels
* execution indicators

ALL components must reflect ONE authoritative state source.
