New onboarding / exchange-connect UX issues found during live testing.

ISSUE #1 — Wrong redirect after disclaimer flow
Current behavior:

* User attempts exchange connection
* Disclaimer appears ✅
* User accepts disclaimer ✅
* User receives “Congratulations / paper trading enabled” message
* Then gets redirected into the mobile app/profile experience instead of remaining in the trading dashboard

This is incorrect.

Expected behavior:
After disclaimer acceptance:

* user should remain inside:
  `https://trade.aicandlez.com/portal`
* OR proceed directly into:
  exchange connection onboarding

The user should NEVER be redirected into the unrelated app/profile screen when attempting exchange setup.

Please inspect:

* disclaimer success redirects
* onboarding redirect handlers
* post-consent navigation
* membership activation callbacks
* exchange connect CTA handlers

Likely issue:
The disclaimer flow is reusing generic onboarding/app navigation instead of trading-dashboard-specific routing.

Correct destination after disclaimer:

```txt
/portal
```

or:

```txt
/settings/exchanges
```

NOT:

```txt
/app/profile
/mobile-app
```
