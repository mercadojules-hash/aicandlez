BUG #1 — Exchange modal closes into wrong page/state

* User goes to:
  `https://trade.aicandlez.com/portal`
* User clicks:
  `Connect Exchange`
* User closes modal using:
  `X`
* Instead of returning cleanly to the live trading portal, the UI redirects/shifts into another screen/state (`/settings/exchanges` style profile/settings screen)

Expected behavior:

* Clicking `X` should:

  * close the modal ONLY
  * preserve current route/state
  * keep user on:
    `/portal`
  * not navigate anywhere
  * not push browser history
  * not mount profile/settings views

Please inspect:

* modal close handlers
* router pushes/navigates
* URL state sync
* browser history manipulation
* exchange modal mount/unmount logic

Likely issue:

* modal is tied to route navigation instead of local UI state

Desired behavior:

```ts
setExchangeModalOpen(false)
```

NOT:

```ts
navigate("/settings/exchanges")
```
